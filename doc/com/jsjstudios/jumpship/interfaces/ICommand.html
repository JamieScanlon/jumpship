ICommand Interface.

In the JumpShip framework, ICommand has two responsibilities, execute an operation and return a result. ICommand's are assumed to be asynchanous and to have a result that gets returned to a function given in the execute() method. Each ICommand instance may be given a unique id to be passed back to the return function. Each ICommand instance may also get an Object containing values it may need to perform it's operation.

public function onResult(resultObj:Object = null):void

Process the ICommand result.

It is anticipated that ICommand's are asynchronous, in which case this is the event handler for a result. The implementation of this handler should call the returnFunc given in the execue() method, passing the resultObj and the id (also given in the execute() method) as arguments in the form:

returnFunc(resultObj);

THIS SHOULD BE THE LAST FUNCTION CALLED BY ANY COMMAND. Even if this Command does not call a service asynchronously. Parameters

resultObj:Object (default = null) — an optional Object containing the result parameters.
