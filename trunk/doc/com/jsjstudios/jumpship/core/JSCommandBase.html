Command Base Class. In this framework commands have the ability to cascade or call other commands. Therefore this base class contains a lot of the same methods as the Controller base class. But whereas the Controller MUST implement primitive command functions, commands are not required to.

addCommands()method 
protected function addCommands():void

A method that is called to add child Commands that this Command can call in the course of its duty's. The format for adding a Command should be as follows:

addCommand ( "getPersonListCommand", GetPersonListCommand );

"getPersonListCommand" is the name of the command that will returned with the result of the child Command.

execute()method 
public final function execute(params:Object = null, returnFunc:Function = null, token:Object = null):void

Executes the JSCommandBase.

This implementation of ICommand is purposefully strict, requiring the returnFunc and params arguments to be present. This method is final and calls the onExecuteOperation() method to carry out the main functionality. Parameters

params:Object (default = null) — an Object containing variables this Command may use to carry out it's task.
 
returnFunc:Function (default = null) — an Function that get's called once the Command has completed.
 
token:Object (default = null) — an optional Object passed back through the argument of the returnFunc.

Throws
— If either params or returnFunc is null.

executeOperation()method 
public function executeOperation():void

The main execution block for a JSCommandBase.

This primitive method is called by the execute() method. After doing it's work (asynchonously or not), the onResultOperation() method must get called with the result of the operation (even if null). Doing so completes the loop of creating a command and returning a result and since commands are logged in JumpShip, it ensures this command does not stay in memory after it has completed it's task.

onResult()method 
public final function onResult(resultObj:Object = null):void

Process the JSCommandBase result.

This handler packages the resultObj into a wrapper Object in the form {type:COMMANDNAME, result:resultObj, id:token} (where token is the token that was passed into the execute() function), and sends the result to the returnFunc. Parameters

resultObj:Object (default = null) — An Object containing the event parameters.
onResultOperation()method 
public function onResultOperation(theResult:Object = null):void

The main result processing block for a JSCommandBase.

This method must be called when this command has completed it's operations. It must also call onResult(theResult) to notify the owner that this command has completed. Parameters

theResult:Object (default = null) — an Object containing the result of the operation (if any).

public final function onStatus(statusObj:Object = null):void

Process the JSCommandBase status.

It is anticipated that Commands may call services, in which case this is the event handler for a Status message. This handler calls the onStatusOperation() Hook Function. Parameters

returnCommandResult()method 
public final function returnCommandResult(theResultObject:Object):void

Processes the result of a JSCommandBase instance.

This is the default returnFunc given to a new JSCommandBase. This method calls the afterFilter() hook method. This method then calls the processCommandResult primitive method with the result. Parameters

theResultObject:Object — An object containing the result information from the command.
